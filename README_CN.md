# Memos — AI 智能体记忆系统

> **高速存储 + 自主记忆进化。**

Memos 将 Memvid 极致的单文件向量存储引擎与 ClawMem 的认知智能引擎相结合——自主决策提取、因果图谱遍历、A-MEM 自进化、以及反馈驱动的衰减机制。采用 Rust 编写以保证极致性能。

## 状态

**规划/架构阶段** — 规格已冻结，实现脚手架正在搭建中。

## 这个项目是什么

Memos **不是**任何一个上游项目的分支。它是一个**全新架构**：

1. 采用 Memvid 的 `.mv2` 单文件格式和 HNSW/BM25 检索引擎作为**存储和检索后端**
2. 采用 ClawMem 的认知层概念——决策提取、因果图谱、A-MEM 进化、内容类型半衰期——作为**记忆智能层**
3. 在两个设计的空白处添加原创研究

## 核心设计原则

| 原则 | 理由 |
|------|------|
| **存储是"哑"的，认知是"聪明"的** | Memvid 处理比特；Memos 处理意义 |
| **只追加是神圣的** | 记忆永不删除，只衰减/归档 |
| **面向智能体，而非面向 RAG** | 记忆是为智能体服务的，不是为文档检索管道服务的 |
| **默认零外部服务** | 单二进制文件，单文件，完全离线可用 |
| **可证明的正确性** | 混合搜索 + 因果图谱 + 反馈信号 |

## 存储层（Memvid）

- **文件格式**：`.mv2` — 单一文件包含 WAL、数据段、FTS 索引、HNSW 索引、时序索引、TOC
- **无外部数据库**：无需 PostgreSQL、SQLite、Redis、Qdrant
- **只追加**：新记忆作为不可变帧追加；旧的标记为衰减/归档，永不覆盖
- **崩溃安全**：已提交的 WAL 帧在突发关闭后仍可恢复

## 认知层

### 决策提取

会话停止时，本地 LLM 观察 transcript 并提取：
- `type`：decision / observation / preference / fact
- `title`：简短描述
- `facts`：结构化键值对
- `narrative`：自由文本摘要
- `contradicts`：此决策所取代的先前决策引用

### 因果图谱

- 节点：记忆片段
- 边：cause → effect（从决策观察中提取）
- 边权重：置信度（0.0–1.0）
- 遍历：MPFP（多路径事实传播），带前向推送

### A-MEM（自适应记忆进化）

每次索引操作时：
1. 提取关键词（3–7 个具体术语）
2. 提取标签（3–5 个通用类别）
3. 生成上下文描述（1–2 句话）
4. 查找相关记忆 → 创建语义链接
5. 如果链接更改，更新相邻元数据

### 反馈循环

- **被引用** → 置信度增强
- **被展示但被忽略** → 衰减信号
- **从未被展示** → 纯时序衰减

### 内容类型半衰期

| 类型 | 基线 | 半衰期 | 衰减规则 |
|------|------|--------|---------|
| `decision` | 0.85 | ∞ | 永不衰减 |
| `hub` | 0.80 | ∞ | 永不衰减 |
| `antipattern` | 0.75 | ∞ | 永不衰减 |
| `research` | 0.70 | 90 天 | 线性衰减 |
| `project` | 0.65 | 120 天 | 线性衰减 |
| `handoff` | 0.60 | 30 天 | 快速衰减 |
| `progress` | 0.50 | 45 天 | 线性衰减 |
| `note` | 0.50 | 60 天 | 线性衰减 |

## 技术栈

| 组件 | 技术 | 用途 |
|------|------|------|
| 存储引擎 | Memvid | `.mv2` 单文件，向量 + 全文 |
| 全文搜索 | Tantivy (via Memvid) | BM25 |
| 向量索引 | HNSW (via Memvid) | 近似最近邻 |
| 关系数据库 | SQLite | 因果边、实体、关系 |
| LLM 推理 | 本地 GGUF / API | 决策提取、因果推理 |
| 嵌入 | BGE-small / 本地 API | 向量生成 |

## 开始使用

```bash
# 安装 Rust（如果尚未安装）
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 克隆并编译
cargo clone https://github.com/zinsser-1686/memos
cd memos
cargo build --release

# 运行测试
cargo test
```

## CLI 命令

```bash
memos init                    # 初始化 vault
memos put <内容>              # 添加记忆
memos search <查询>           # 混合搜索
memos query <查询>            # 完整管道
memos intent <查询>           # 意图分类搜索
memos timeline <文档ID>       # 时间邻域
memos causal <文档ID>         # 因果链遍历
memos extract                 # 从 transcript 提取决策
memos reflect                 # 运行 A-MEM 整合
memos status                  # vault 统计
memos doctor                  # 健康检查
```
